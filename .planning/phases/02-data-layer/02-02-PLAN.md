`data/applications.md` (real format — verify by reading first 10 lines):
```
# Applications Tracker

| # | Date | Company | Role | Score | Status | PDF | Report | Notes |
|---|------|---------|------|-------|--------|-----|--------|-------|
| 126 | 2026-05-13 | Typeform | Product Designer - Integration | 3.65/5 | Evaluated | ❌ | [126](reports/126-typeform-pd-integration-2026-05-13.md) | EU remote... |
```

Parsing rules for applications:
- Skip header line(s) and the `|---|---|...` separator line
- A valid row matches: `| {num} | {YYYY-MM-DD} | {company} | {role} | {score}/5 | {Status} | {✅ or ❌} | {[num](reports/path.md) or empty} | {notes} |`
- score "3.65/5" → number 3.65; "/5" suffix is stripped
- pdf "✅" → true, "❌" → false (any other emoji → false + ParseError on that row)
- reportPath: extract URL from `[num](path)` or null if empty
- Per-row error boundary: try/catch around each row's parse+validate; bad row → push to errors[], continue

`data/pipeline.md` (real format — checkbox list, NOT a table):
```
# Pipeline — Pending Evaluations

## Pendientes

<!-- comment -->
- [x] #050 | https://jobs.ashbyhq.com/quora/c24fe257-bf50-4a63-8c3c-a93c2685d0e8 | Quora (Poe) | Lead Product Designer - Design Systems | 4.2/5 | PDF ❌
- [s] https://jobs.ashbyhq.com/rundoo/77e75f8c-... | Rundoo | Design Systems Designer | SKIPPED: location=Redwood City hybrid
- [!] https://jobs.ashbyhq.com/n8n/fe05c209-... | n8n | Senior/Staff Product Designer — Error: posting not in active Ashby JSON
- [ ] https://example.com/new-listing | Acme | Senior Designer
```

Parsing rules for pipeline:
- Process ONLY lines starting with `- [x]`, `- [s]`, `- [!]`, or `- [ ]`
- Ignore comments, headings, blank lines, and any other content
- state mapping: `x`→'evaluated', `s`→'skipped', `!`→'error', ` `→'pending'
- After the checkbox, split on ` | ` (pipe with spaces) to get fields
- First non-empty token that matches `#NNN` → set num (optional)
- First token matching `https?://...` → url (REQUIRED — if none, push ParseError and skip)
- Remaining tokens: company, title, score (`N.N/5` or `N/5`), pdf indicator (`PDF ✅`/`PDF ❌`)
- Skipped entries have a single note field instead of structured columns; preserve as note, leave company/title/score/pdf null

`reports/NNN-{slug}-{YYYY-MM-DD}.md` (real format):
```
# Senior Product Designer - Design Systems — Hostaway

**Score:** 4.35/5
**URL:** https://careers.hostaway.com/...
**PDF:** ❌
**Legitimacy:** High Confidence
**Date:** 2026-04-21

## Blocks

| Block | Score | Notes |
|-------|-------|-------|
| A — CV Match | 4.0 | DS owner role... |
| B — North Star | 5.0 | Role title IS "Design Systems" |
| C — Comp | 3.0 | "Competitive + equity" |
| D — Culture | 5.0 | 100% remote EMEA |
| E — Red flags | 5.0 | No blockers |
| **F — Global** | **4.35** | |

## Match analysis
{body...}
```

Parsing rules for reports:
- Filename pattern: `^(\d{3})-(.+)-(\d{4}-\d{2}-\d{2})\.md$` → captures num, slug, date
- Title is the first `# ` H1 (entire line text after `# `)
- Header `**Key:** value` pairs (Score, URL, PDF, Legitimacy) — Score "4.35/5" → 4.35, PDF ✅→true ❌→false, URL nullable, Legitimacy free-text
- `## Blocks` table — match rows by regex: `^\|\s*(\*\*)?([A-F])\s*[—-]\s*(.+?)(\*\*)?\s*\|\s*(\*\*)?([\d.]+)(\*\*)?\s*\|\s*(.*?)\s*\|$`
- Each Block A-F either parsed (`{ score, notes }`) or null if missing in table
- `body` = everything after the `## Blocks` section (or full content if no Blocks)
- PDF companion: if `{outputDir}/{num}.pdf` exists, OVERRIDE the header's pdf flag to true (file system is truth for v1 — `pdf` field in Report schema reflects companion existence). If outputDir is undefined, use the header value only.
- gray-matter is used to strip any YAML front-matter if present (most reports don't have it; tolerate either)
